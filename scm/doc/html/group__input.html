Algorithm

  • Open the case input file found in the processed_case_input dir corresponding to the experiment name.
  • Get the dimensions (global group).
  • Allocate the dimension variables.
  • Read in the dimension variables.
  • Read in the initial conditions.

    • Allocate the initial profiles.
    • Read in the initial profiles. The variable names in all input files are expected to be identical.
  • Read in the forcing data.
    • Find group ncid for forcing group.
    • (Recall that multidimensional arrays need to be read in with the order of dimensions reversed from the netCDF file).
    • Allocate the time-series and 2D forcing data.
    • Read in the time-series and 2D forcing data.

Subroutine to get basic model configuration data from a namelist file (whose name is specified as the first argument on the command line) and from data entered on the command line with the format: "var1='string' var2=d.d var3=i". Namelist variables listed on the command line override those that are specified in the external namelist file. The case configuration namelist variables are also written out to a namelist file placed in the output directory. Note: This routine uses GET_COMMAND which is an intrinsic routine in the Fortran 2003 standard. This requires that the compiler supports this standard.

Parameters
[out]experiment_namename of the experiment configuration file (usually case name)
[out]model_namename of the host model (currently only GFS supported)
[out]physics_suitename of the physics suite name (currently only GFS_operational supported)
[out]case_namename of case initialization and forcing dataset
[out]dttime step in seconds
[out]runtimetotal runtime in seconds
[out]output_frequencyfreqency of output writing in seconds
[out]swrad_frequencyfreqency of calling SW radiation in seconds
[out]lwrad_frequencyfreqency of calling SW radiation in seconds
[out]n_levelsnumber of model levels (currently only 64 supported)
[out]time_scheme1 => forward Euler, 2 => filtered leapfrog
[out]output_dirname of the output directory
[out]output_filename of the output file (without the file extension)
[out]thermo_forcing_type1: "revealed forcing", 2: "horizontal advective forcing", 3: "relaxation forcing"
[out]mom_forcing_type1: "revealed forcing", 2: "horizontal advective forcing", 3: "relaxation forcing"
[out]relax_timerelaxation time scale (s)

Algorithm

Define default values for case configuration (to be overridden by external namelist file or command line arguments)

Parse the command line arguments.

  • Get the entire command used to invoke the SCM (including arguments)
  • Discard SCM invocation command.
  • Get experiment name from first argument (if not present, use default values in this source file)
  • Put remaining command line arguments into "internal" namelist to read in after the external file.

Attempt to read in external namelist file.

"Read" in internal namelist variables from command line.

Check for namelist reading errors. If errors are found, output the namelist variables that were loaded to console and ask user for input on whether to proceed.

If there was an error reading from the command line, use the default values of the namelist variables and check with the user whether to continue.

Write the case_config namelist variables to an output file in the output directory for inspection or re-use.
